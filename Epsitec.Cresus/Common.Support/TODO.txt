
Ce qu'il reste à faire dans "Support"
-------------------------------------

-	Les exceptions (pas seulement celles définies dans Common.Support)
	devraient toutes être marquées comme étant [Serializable] afin de
	permettre leur utilisation dans des contextes de "remoting".
	
	Voir à ce sujet l'article :
	
	ms-help://MS.VSCC.2003/MS.MSDNQTR.2003FEB.1033/cpref/html/frlrfSystemExceptionClassctorTopic3.htm
	
-	Il faut faire de même pour les attributs :
	
	o	BundleAttribute
	o	CommandAttribute
	o	SuppressBundleSupportAttribute
	

-	Dans ImageProvider.cs, ajouter une REGEX pour vérifier que le nom
	de l'image (si le protocole file: est utilisé) ne contient que des
	caractères valides (pas de \ ni de /, ni de ..).


-	La classe ObjectBundler ne contient pas encore le code qui
	énumère toutes les assemblies chargées et les analyse
	dynamiquement pour enregistrer tous les types qui implémentent
	l'interface IBundleSupport.


-	La classe Implementation.BaseProvider ne fait pour l'instant aucun
	accès à la base de données. Il faut implémenter les méthodes qui
	manquent, en particulier compléter GetData et Setup.

RESSOURCES DANS LA BASE DE DONNEES
-----------------------------------------------------------------

La base de données des ressources d'une application se nomme :

	application.resdb

(avec éventuellement l'extension ajoutée par DbInfrastructure et
les autres couches).

En plus des tables décrivant les méta-données, il y a les tables
suivantes :

-	DbTable.Name = "Data";
-	DbTable.Name = "Data.xx";

La première contient toutes les données par défaut (non localisées)
et les autres, avec le suffixe "xx" correspondent aux versions
traduites.

Les colonnes dans une table "Data" sont les suivantes :

-	Clef et statut (CR_ID, CR_REV, CR_STAT).
-	Name (DbColumn.Name = "Name";), type DbTypeString, max=100
-	Data (DbColumn.Name = "Data";), type DbType(DbSimpleType.ByteArray)
	contenant des BLOBs binaires.

La recherche dans une des tables (Data ou Data.xx) se fait au moyen
d'un SELECT tel que :

-	Name = id de la ressource
-	CR_REV = 0
